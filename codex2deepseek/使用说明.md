# Codex 接 DeepSeek 本地代理

这个目录已经整理成一套更稳妥的本地代理方案。

核心原则：

- `auth.json` 只给 Codex 访问本地代理时做本地鉴权
- 真实 DeepSeek key 单独放在 `proxy_settings.json`
- 这样即使把 `config.toml` 和 `auth.json` 复制到 `~/.codex`，也不会把真实 DeepSeek key 塞进 Codex 的 `auth.json`

## 文件说明

- `deepseek_proxy.py`：本地中转服务，默认监听 `127.0.0.1:50010`
- `proxy_settings.json`：代理自己的配置文件，保存 DeepSeek key、默认模型和可选本地代理令牌
- `config.toml`：给 Codex 使用的 provider 配置，指向本地代理
- `auth.json`：给 Codex 使用的本地代理令牌，不是 DeepSeek 的真实 key
- `start_proxy.bat`：Windows 双击启动代理
- `start_proxy.sh`：Linux 或 macOS 启动代理
- `install_codex_config.bat`：把当前目录的 `config.toml` 和 `auth.json` 复制到 `%USERPROFILE%\.codex`
- `install_codex_config.sh`：把当前目录的 `config.toml` 和 `auth.json` 复制到 `~/.codex`

## 当前配置结构

`proxy_settings.json` 里保存真实 DeepSeek 配置：

```json
{
  "deepseek_api_key": "你的 DeepSeek key",
  "default_model": "deepseek-v4-pro",
  "deepseek_base_url": "https://api.deepseek.com",
  "local_proxy_token": "local-deepseek-proxy"
}
```

`auth.json` 里保存的是本地代理令牌：

```json
{
  "OPENAI_API_KEY": "local-deepseek-proxy"
}
```

## 使用步骤

1. 双击运行 `start_proxy.bat`
2. 如果要把 Codex 切到这套代理配置，再运行 `install_codex_config.bat`
3. 重启 Codex
4. 确认 Codex 使用模型 `deepseek-v4-pro`

## 代理读取顺序

- `DEEPSEEK_API_KEY` 环境变量
- `proxy_settings.json`
- `使用方法.txt` 里的 `api_key`

默认模型读取顺序：

- `DEEPSEEK_MODEL` 环境变量
- `proxy_settings.json`
- `使用方法.txt` 里的 `default_text_model`

本地代理令牌读取顺序：

- `DEEPSEEK_PROXY_TOKEN` 环境变量
- `proxy_settings.json`
- `auth.json`

## 已支持能力

- `GET /v1/models`
- `POST /v1/responses`
- 普通文本消息转发
- `Codex tool`(Codex 工具) 的函数调用转发
- `thinking` 模式下的工具回路续传
- 本地合成 `SSE`(服务器发送事件) 流式输出
